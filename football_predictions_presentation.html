Technical Architecture
Layered and modular — easy to extend

Project Structure (high-level)

  • config/ — YAML per environment (prod/dev/test)
  • data/ — raw, processed, external (dev & prod partitions)
  • notebooks/ — exploratory & production notebooks
  • scripts/ — collectors, training, prediction utilities
  • models/ — stored model artifacts by competition

Pipeline Layers

  • Data collection: requests + BeautifulSoup, checkpointing, rate limiting
  • Data processing: cleaning, rolling features, master dataset
  • ML: LightGBM & XGBoost, Optuna HPO, time-series CV
  • Deployment: joblib models, CLI, future API/dashboard
Architecture overview
Data Pipeline
From FBRef scraping to master dataset

Collection layer

  • BeautifulSoup + requests (no browser automation)
  • Rate limits & UA rotation, exponential backoff
  • Outputs: JSON / CSV / Parquet
  • Collectors: fixtures, match_stats, wages, team_id_mapper

Processing layer

  • Cleaning & validation (missing values, types)
  • Feature engineering: 5-match rolling averages, rest days, opponent features
  • Master dataset assembly and consistency checks
  • Data schema with 64+ stats + engineered fields

Core Components
Collectors, mappers, orchestration

Collectors & Mappers

  • team_id_mapper.py — canonicalizes team names across sources
  • fixtures_collector.py — scrapes fixtures & results
  • match_stats_collector.py — detailed match stats
  • wages_collector.py — player -> aggregated team wages

Orchestration

  • run_all_collectors.py — coordinates and checkpoints pipeline runs
  • Logging, error reporting, and retry logic built-in
  • Configurable via YAML per environment

Machine Learning
Models, features & evaluation

Primary model — LightGBM (v2)

  • Task: Multi-class (Win/Draw/Loss)
  • Features: 192 selected features (form, wages, context)
  • HPO: Optuna (100 trials), 5-fold stratified CV
  • Performance: Test acc 49.21%; CV acc 55.86%

Experimental — XGBoost RandomForest (v3)

  • Task: Goal difference regression (capped -3..+3)
  • Metrics: MAE 1.39, R² 0.068
  • Classification via thresholding: ~47.4% accuracy

Performance & Resources
Benchmarks & system needs

Performance

  • Data collection: ~500–1000 matches/hour (rate-limited)
  • Training HPO time: ~15–30 minutes
  • Inference: <1 ms per prediction

System requirements

  • Python 3.8+
  • 8GB+ RAM recommended
  • Storage: 5GB+ for historical data
